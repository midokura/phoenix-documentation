Version 26.1.0 of AI Factory is now available.

## Overview

<!-- truncate -->

### Hedgehog

- **ESLAG and Fabric connection types** *(new feature)*: Two new connection kinds are now supported: ESLAG (EVPN multi-homing, server ↔ 2–4 switches) and Fabric (spine ↔ leaf point-to-point with /31 IPs).
- **Parameterized `networks` and `hostdev`** *(new feature)*: The flat `bridge_name` / `vm_mac` / `mgmt_nic_pci` keys are replaced by a structured `networks` list and `hostdev` dict. Old flat keys remain backwards compatible.
- **Interface renaming removed** *(cleanup)*: The `network.rename_interfaces` feature was dropped. Interface identity is now taken directly from `networks[0].mac.address`.

### Neutron

- **L3 HA failover fixed** *(bug fix)*: A 4–5 minute failover window caused by the keepalived health check interval was resolved by tuning `ha_vrrp_health_check_interval`, `agent_report_interval`, and `agent_down_time`.

### Ceph

- **RGW Keystone integration fully configurable** *(new feature)*: All RGW Keystone parameters (URL, accepted roles, implicit tenants, S3 auth, SSL verification) are now exposed and idempotent.
- **RGW admin user provisioning** *(new feature)*: New task provisions an IaaS Console admin user in the object gateway.
- **PG management simplified** *(cleanup)*: `pg_autoscale_mode` removed. PG count is now auto-calculated from OSD count, replication factor, and pool count.

### Magnum

- **Addon mutator** *(new feature)*: New role that patches Cinder CSI topology annotations in Magnum CAPI cluster resource sets via a CronJob.
- **CAPI kubeconfig encrypted** *(new feature)*: The `encrypt.yml` playbook now covers `kubeconfig` files under `infra-management/*/config/config/magnum/`, preventing them from being committed in plaintext.

### Infrastructure

- **Custom Docker registry authentication** *(new feature)*: New `docker_registry_username` / `docker_registry_password` variables support authenticated private registries.
- **PXE boot: `needrestart` config extended to compute nodes** *(bug fix)*: Previously only applied to `role: control` nodes.
